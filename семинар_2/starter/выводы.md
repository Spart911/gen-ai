# Выводы по генерации заявок на ДПО

## Перекосы и борьба с ними

На первом прогоне (только `seed_city`) города распределились хорошо (стратификация по 12 городам), но специальности и курсы «слиплись»: логопед 30%, «Коррекционная педагогика» 48%, а «Иванова Мария Петровна» встретилась 12 раз из 50. Это классический **mode collapse** — модель тянется к «типичному» педагогическому профилю и популярным русским ФИО.

Для «отлично» добавлена **полная стратификация**:

1. **Города** — по 4–5 заявок на каждый из 12 городов (`stratify()`).
2. **Специальности** — ровно по 5 заявок на каждую из 10 специальностей (10% каждая).
3. **Курсы** — подбираются через `SPECIALITY_COURSE_MAP`: каждой специальности назначается правдоподобный курс с балансировкой счётчиков.
4. **ФИО** — `surname_hint`, список used names, reject дубликатов с retry.

Итог: топ-город 10%, топ-специальность 10%, **50 уникальных ФИО из 50**.

## Влияние стратификации на специальности

Когда квотировали только города, 28 из 50 оказались логопедами/психологами. После квотирования специальностей каждая категория — по 5 заявок; все 8 курсов представлены.

## Валидатор и retry

`@field_validator` на `graduation_year` проверяет согласованность с `age`. Все 50/50 прошли `check_applications.py`. Валидатор, вероятно, срабатывал на промежуточных попытках внутри `max_retries=3`, но финальный ответ всегда проходил. Generator дополнительно делает post-check на city/speciality/course/ФИО.

## Расширенный анализ

Семинарский **`5_analysis.py` не менялся**. Подготовка и запуск одной командой:

```bash
python prepare_analysis_data.py
```

Маппинг в `analysis_input.json`: `full_name→name`, `speciality→occupation`, `years_of_experience×15000→income_rub` (прокси для boxplot).

**Результаты:** `report.md`, `crosstab_city_speciality.csv`, `ages.png`, `occupations.png`, `income_by_occupation.png`.

**2–3 спорные комбинации** (подробно в `report.md`):

1. Социальный работник → Менеджмент в образовании (Кузнецов, Самара).
2. Экономист → Бухгалтерский учёт (Никитин, Санкт-Петербург).
3. Васильев Игорь Петрович — стаж 10 лет при возрасте 30 (Pydantic не ловит).

## Что осталось нерешённым

Стратификация жёстко задаёт поля — хорошо для распределения, но убирает «естественный» выбор модели. `5_analysis.py` видит город×специальность; пары специальность→курс нужно смотреть отдельно в `applications.json`.
