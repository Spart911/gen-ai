# Сравнение стратегий чанкинга (п. 4 ДЗ)

**Корпус:** 12 документов ru.stackoverflow.com, 155 938 символов  
**Gold:** 15 вопросов  
**Retrieval:** hybrid (ChromaDB dense + BM25 + RRF), метрика **hit-rate@5**  
**Эмбеддер:** `paraphrase-multilingual-MiniLM-L12-v2`

## Стратегии

| | **A — fixed** | **B — recursive** |
|---|---------------|-------------------|
| Алгоритм | `text[i : i + 2000]`, без overlap | `RecursiveCharacterTextSplitter(400, overlap=80)` |
| Separators | — | `###`, `##`, абзацы, предложения |
| Обоснование | Baseline из задания | Markdown-треды SO: режем по заголовкам тредов/ответов, overlap сохраняет контекст на границах |

## Результаты hit-rate@5

| Конфигурация | fixed | recursive |
|--------------|------:|----------:|
| **Hybrid (dense + BM25 + RRF)** | **1.00** (15/15) | **1.00** (15/15) |
| Dense-only (baseline) | 0.97 (14.5/15) | 0.97 (14.5/15) |
| Чанков в индексе | 85 | 594 |
| Средний ранг gold-дока в top-5 | **1.17** | 1.72 |

По основной метрике (**hybrid hit-rate@5**) стратегии **сравнялись**. Различия видны в размере индекса и качестве ранжирования.

## Где стратегии ведут себя по-разному

### Q1 — multi-hop (единственный провал dense-only)

**Вопрос:** чат-бот по базе знаний + память диалога  
**Gold:** `doc_01` + `doc_03`

| Retrieval | fixed | recursive |
|-----------|-------|-----------|
| Hybrid | 1.00 — оба документа в top-5 | 1.00 — оба документа в top-5 |
| Dense-only | **0.50** — только `doc_01` (4 из 5 чанков) | **0.50** — только `doc_03` (5 из 5 чанков) |

**Что происходит:** dense «залипает» на одном семантически близком документе и не подтягивает второй источник multi-hop. BM25 через ключевые слова (`ChromaDB`, `messages`, `база знаний`) вытягивает недостающий документ — hybrid закрывает вопрос для обеих стратегий чанкинга.

### Ранжирование при одинаковом hit-rate

При hybrid=1.00 **fixed** ставит gold-документы ближе к top-1 (avg rank **1.17** vs **1.72**): крупные чанки (2000 симв.) содержат больше ключевых слов запроса в одном эмбеддинге. **Recursive** даёт более точные фрагменты (отдельный тред/ответ), но gold-документ чаще оказывается на 2–3 позиции.

## Вывод

**По hit-rate@5 при hybrid-поиске победителя нет — 1.00 у обеих.** Для production на этом корпусе разумнее **recursive**: чанки совпадают со структурой Markdown-тредов, цитаты точнее, цена — индекс в ~7× больше (594 vs 85 чанков). **Fixed** выигрывает компактностью индекса и чуть лучшим ранжированием, но режет фразы посередине абзаца.

Критичнее стратегии чанкинга оказался **гибридный retrieval**: без BM25 multi-hop Q1 падает до 0.50 при любой нарезке.
